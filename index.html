Gizmos

http://wiki.blender.org/index.php/Doc:2.4/Manual/3D_interaction/Transform_Control/Manipulators

Gizmos sind 3D-Objekte innerhalb der 3D-Szene die dazu dienen die Szene selbst du beeinflussen und zu ändern. In Blender werden diese Manipulatoren genannt. Hier zu sehen sind:
  1. Translations-Manipulator
  2. Skalier-Manipulator
  3. Rotations-Manipulator
  4. Alle 3 Zusammen

Gizmos

  • PlaneSensor
  • SphereSensor
  • CylinderSensor
Alle Sensoren wandeln Mausbewegungen und Touchgesten in Rotationen oder Translationen um. Im Falle des PlaneSensors in Tranlationen, im Falle des SphereSensors in eine freie rotation und im Falle des CylinderSensors in eine Rotation in einer Ebene.

Gizmos

http://doc.x3dom.org/tutorials/animationInteraction/transformations/index.html

Hier ist ein Beispiel von CylinderSensor und PlaneSensor Knoten. Jedes Band ist der Geschwisterknoten eines CylinderSensors. Der Zylinder mit den Kegeln ist der Geschwisterknoten eines PlaneSensors.

<Group>

    autoOffset='true'

    >

Der PlaneSensor funktioniert indem er auf alle Touch und Mouseevents seiner Geschwisterknoten reagiert. In diesem Fall ist es die Transformation und alles was da drin enthalten ist. Der PlaneSensor hat einige Attribute wie minimale und maximale Auslenkung, sowie einen Callback. Dieser Callback erhält die jeweilige Auslenkung des Sensors, diese Auslenkung kann dann auch auf andere Objekte angewandt werden.

Gizmos

Callback



const processTranslationGizmoEvent = event => {
  if (event.fieldName === 'translation_changed') {
    //convert the sensor's output from sensor coordinates to world coordinates (i.e., include its 'axisRotation')
    const sensorToWorldMatrix = x3dom
      .fields.SFMatrix4f.parseRotation(event.target.getAttribute("axisRotation"));

    const translationValue = sensorToWorldMatrix.multMatrixVec(event.value);

    //transform the affected sensor geometry
    document.getElementById('translationHandleTransform')
      .setFieldValue('translation', translationValue);
    document.getElementById('rotationHandleTransform')
      .setFieldValue('translation', translationValue);

    //transform the affected element
    document.getElementById('teapotTranslation')
      .setFieldValue('translation', translationValue);
  }
}

function drop (event) {
  const files = flatten(event.dataTransfer.files) // turn into real array

  files.forEach(file => {
    const reader = new window.FileReader()

    reader.onload = event => {
      Project
        .uploadFile(file.name, event.target.result)
        .catch(() => window.alert('Upload failed.'))
        .then((result) => {
          this.props.data.appendChild(renderJSX(
            <transform>
              <inline
                url={result.data}
                namespacename={file.name}
                />
            </transform>
          ))
        })
    }
    reader.readAsText(file)
  })
}

// much simplified
import React from 'react'
import {setUpInline} from 'x3domUtils'
import {throttle} from 'lodash'
import {flatten} from 'ramda'
import TreeView from 'TreeView'

const watchedAttributes = [
  'diffuseColor', 'orientation', 'position',
  'render', 'rotation', 'scale', 'translation'
]
const rerender = throttle(React.render, 100)
const container = document.querySelector('#container')
const x3dNode = document.querySelector('x3d')
const x3dObserver = new window.MutationObserver(mutations => {
  mutations.forEach(mutation => {
    if (mutation.type === 'attributes') {
      if (watchedAttributes.some(name => name === mutation.attributeName)) {
        rerender(<TreeView x3dNode={x3dNode} />, container)
      }
    } else if (mutation.type === 'childList') {
      flatten(mutation.addedNodes)
        .filter(node => node.nodeName.toLowerCase() === 'transform')
        .forEach(transform => {
          const inline = transform.children[0]

          if (inline && inline.nodeName.toLowerCase() === 'inline') {
            setUpInline(inline, x3dNode)
          }
        })

      rerender(<TreeView x3dNode={x3dNode} />, container)
    }
  })
})

x3dObserver.observe(x3dNode.querySelector('scene'))

Aussicht

  • Portierung zu Electron
    • Offline
    • Performanter
    • Bearbeitung größerer Daten
    • Vereinfachung der Codestruktur
    • Kompletter Wegfall des Servers
  • Electron wurde von GitHub entwickelt um einen code editor mittels Webtechnologien zu implementieren. Ganz einfach ausgedruckt ist Electron ein gekapselter Chrome Browser zusammen mit einer Node.js instanz, welche beide über Inter-process Communication miteinander kommunizieren können. Die GUI wird mittels HTML (oder einer Template Engine die HTML generiert) beschrieben oder dynamisch generiert. Im prinzip geht alles was auch für Webseiten geht. Dabei kann alles mit Electron mit allen Assets gebundlet und ausgeliefert werden. Das ermöglicht es entweder Desktop Applikationen mittels Webtechnologien zu entwickeln die auch ohne Interverbindung funktionieren, als auch Webapplikationen (wie SceGraToo) zu einer eigentständing Desktopvariante zu portieren.
  • Größere Datein -> unabhängig von Up- und Downloadgeschwindigkeit
  • Performanter -> Ladezeiten verkürzen sich -> Kein Roundtrip zum Server
  • Vereinfachung der Codstruktur -> Dateisystem API einfacher ist als die Netwerk API
  • Kompletter wegfall des Servers
